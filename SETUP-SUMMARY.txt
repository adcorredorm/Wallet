HOW HOT-RELOAD WORKS
================================================================================

FRONTEND (Vite HMR):
1. You edit: frontend/src/components/Card.vue
2. Docker volume detects change
3. Vite watches files and detects change
4. Vite recompiles only changed module
5. HMR sends update via WebSocket (ws://localhost:5173)
6. Browser applies update
7. You see changes in ~1-2 seconds
8. Application state preserved

BACKEND (Flask Auto-Reload):
1. You edit: backend/app/routes/api.py
2. Docker volume detects change
3. Flask's reloader detects change
4. Flask restarts application context
5. Watch for: "[Restarting with stat]" in terminal
6. Next API call uses new code
7. No container restart needed

DATABASE:
1. Data persists in docker volume
2. Survives service restarts
3. Migrations run automatically on startup
4. Access via pgAdmin at http://localhost:5050

TROUBLESHOOTING QUICK REFERENCE
================================================================================

ISSUE: Changes not appearing
SOLUTION:
  - Frontend: Clear browser cache (Cmd+Shift+Delete)
  - Backend: Check Flask reloader in logs
  - Docker: Restart service (docker-compose restart frontend)

ISSUE: Port already in use
SOLUTION:
  - Find process: lsof -i :5173
  - Kill it: kill -9 <PID>

ISSUE: Services won't start
SOLUTION:
  - Check logs: docker-compose logs
  - Validate config: docker-compose config > /dev/null
  - Rebuild images: docker-compose build --no-cache

ISSUE: Database connection error
SOLUTION:
  - Check DB status: docker-compose ps db
  - View DB logs: docker-compose logs db
  - Restart DB: docker-compose restart db

For more troubleshooting:
→ See DEVELOPMENT-WORKFLOW.md (Troubleshooting section)
